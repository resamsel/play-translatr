Aktivität

Die Aktivität zeigt an, wie aktiv Translatr genutzt wird. Dabei wird die Änderungshistorie genutzt, in der sich Beiträge für Benutzer, Projekte, Sprachen, Schlüssel, und Übersetzungen befinden. So kann die Aktivität eines Projekts bestimmt werden, oder auch die eines Benutzers.

Command Line Interface

Das Command Line Interface ermöglicht die Verwaltung der Übersetzungen über die Kommandozeile. Die Operationen können dadurch in Build-Skripten genutzt werden, um automatisiert aktuelle Übersetzungen zu erhalten.

Installation

Füge folgendes im Terminal ein:

curl -fsSL https://raw.githubusercontent.com/resamsel/translatr/master/install.sh | bash

Nutzung

Um Translatr CLI nutzen zu können, brauchen wir eine .translatr.yml Konfigurationsdatei. Lege diese Datei im Verzeichnis an, für das CLI werden soll. Translatr benutzt diese Datei als Konfiguration.

Beispiel .translatr.yml für Play messages
translatr:
  endpoint: http://localhost:9000/
  access_token: <Zugangsschlüssel>
  project_id: <Projekt-ID>
  pull:
    file_type: play_messages
    target: conf/messages.?{{ '{' }}locale.name}
  push:
    file_type: play_messages
    target: conf/messages.?{{ '{' }}locale.name}

Lege einen neuen Zugangsschlüssel (Access-Token) an und trage den Schlüssel in .translatr.yml ein: Zugangsschlüssel.

Danach lege ein Projekt über die Weboberfläche an und trage die Projekt-ID (40 Zeichen UUID in der URL, z.B. http://localhost:9000/project/6f1ee0a7-b5d1-4c7f-8e4d-46c09fece220 -> Projekt-ID: 6f1ee0a7-b5d1-4c7f-8e4d-46c09fece220) ein.

Hochladen

Mit der push-Operation werden die passenden messages-Dateien zum angegebenen Endpunkt hochzuladen. Sprachen werden dabei angelegt, falls notwendig.

translatr push

Dadurch werden bestehende Nachrichten mit den Werten in den messages-Dateien überschrieben. Existierende Schlüssel und Sprachen bleiben bestehen, diese Operation ist keine Synchronisierung.

Herunterladen

Mit der pull-Operation werden alle Sprachen in die konfigurierten Dateien heruntergeladen (Konfiguration: translatr.pull.target).

translatr pull

Dadurch werden bestehende lokale Dateien überschrieben. Alle bekannten Sprachen werden heruntergeladen.
